## Decisions Made

- **Localização dos headers:** `proxy.ts` (não `next.config.ts`). Justificativa: `proxy.ts` já está em uso para `updateSession` (auth), o matcher cobre todas as rotas dinâmicas (incluindo `/r/[slug]`), e atribuir headers a `response.headers` após `updateSession` garante que **redirects** (login, /, /opportunities) também recebam headers — o que `next.config.ts headers()` cobre, mas com o efeito colateral de duplicar configuração. Decisão locked: `next.config.ts` fica reservado para Plan 06 (`withBotId`).
- **CSP `'unsafe-inline'` em script-src:** aceito como tech debt MVP — locked em CONTEXT.md decisão A6. Justificativa: Next.js 16 inline hydration scripts não funcionam sem `'unsafe-inline'` (ou nonce). Nonce-based CSP requer middleware injection e desliga static optimization. Apertar fica para Phase 8+ caso cliente piloto exija.
- **`.returns<T>()` posicionado AO FINAL da chain:** quando aplicado logo após `.select(string)`, o builder vira `PostgrestTransformBuilder` e perde `.eq()/.or()/.order()`. Solução: chamar `.returns<T>()` no `await` final (`await q.returns<Opportunity[]>()`) ou diretamente após a sequência de filtros/order. Inferência de tipos preservada sem `as Opportunity[]` em userland.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Supabase type inference perde Row shape com `.select(string-variável)`**
- **Found during:** Task 2 (queries.ts whitelist)
- **Issue:** Ao substituir `.select('*')` por `.select(OPPORTUNITY_COLUMNS)` (constante string), o TypeScript do Supabase narrowa o tipo para `GenericStringError[]` em vez do `Row` da view. `npm run typecheck` falhou com 3 erros TS2322 ("Type 'GenericStringError[]' is not assignable to type ...").
- **Fix:** Adicionar `.returns<Opportunity[]>()` / `.returns<Opportunity>()` / `.returns<OpportunityPhase[]>()` ao final de cada chain (após filtros/order, no `await` ou logo antes). Não impacta runtime — só re-instrui o TS.
- **Files modified:** lib/opportunities/queries.ts
- **Verification:** `npm run typecheck` passa clean após o ajuste.
- **Committed in:** 17e2272 (Task 2 commit — fix incluso no mesmo commit do refactor)
