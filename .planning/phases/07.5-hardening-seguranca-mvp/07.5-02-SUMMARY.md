## Decisions Made

- **Apply manual em vez de `supabase db push`:** decidido em execution context — projeto roda em Supabase Cloud, não local Docker. Usuário aplica via Dashboard SQL Editor para manter controle visual de mudanças no schema.
- **Helper `tenantSequencesTable()` untyped no teste:** `lib/database.types.ts` ainda não conhece `tenant_sequences` (depende do user rodar `npm run gen:types` PÓS apply). Cast localizado no teste evita bloqueio sem mascarar erros em outras consultas.
- **`describe.skipIf` + lazy init:** o corpo do describe roda mesmo em skip mode (Vitest só pula os `it`s). Sem lazy init, `const sb = serviceRoleClient()` no top-level levanta `NEXT_PUBLIC_SUPABASE_URL ausente`. Solução: declarar `let sb` fora e atribuir dentro de `beforeAll`.
- **Trigger sempre-override (sem `if new.seq_id is null`):** plano original sugeriu manter o `if null`, mas a recomendação de RESEARCH.md (Pattern 2 linha 394) e CONTEXT.md são claras — `seq_id` é server-derived, cliente nunca define. Confirma também o threat T-07.5-C-02 (tampering).
