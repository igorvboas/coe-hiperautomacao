## Decisions Made

- **`.strict()` por variant, não no baseSchema** — Zod 4.x exige que `.extend()` opere sobre object schema sem `unknownKeys: 'strict'` ainda. `.strict()` aplicado depois de cada `.extend()` retorna o object final com check ativo. Decisão alinhada com RESEARCH.md §Pattern 1 line 242.
- **Limite de 8KB para `formulario_extras` via `.superRefine` (não `.refine`)** — `.superRefine` permite emitir issue com path correto e código `'custom'`; `.refine` retorna apenas boolean. Mensagem pt-BR: "formulario_extras excede 8KB serializado".
- **Rule 2 no `updateOpportunity`** — adicionar `.eq('tenant_id', profile.tenant_id)` mesmo com RLS já protegendo. Custo: +1 query (auth.getUser + profile fetch). Benefício: defesa em profundidade explícita, intent claro no código, protege contra regressão futura de RLS policy. Pattern alinhado com `createOpportunity`.
- **NÃO tocar em `createPublicOpportunity`** — escopo explícito do Plan 06 (Bloco D). Apenas comentário no PLAN-03 confirmando que foi auditado e deixado intacto.
