### `supabase/migrations/0006_seq_id_atomic.sql` (migration — exact analog)

**Analog:** `supabase/migrations/0001_init.sql` lines 254–272 (trigger atual a substituir).

**Migration header pattern** (das migrations existentes — `0001`, `0004`, `0005`):

```sql
-- =============================================================================
-- 0006_seq_id_atomic.sql — Atomicidade no seq_id por tenant
-- =============================================================================
-- Substitui o trigger set_opportunity_seq_id() de 0001_init.sql, que usa
-- `select max(seq_id)+1` (race-prone), por uma versão atômica baseada em
-- tabela auxiliar `tenant_sequences` + UPDATE ... RETURNING (row-lock).
--
-- Pré-requisito: 0001_init.sql aplicado.
-- =============================================================================

set session characteristics as transaction read write;
set default_transaction_read_only = off;
set check_function_bodies = off;
```

**Trigger function pattern atual** (`0001_init.sql` lines 255–268 — o que será substituído):

```sql
create or replace function set_opportunity_seq_id()
returns trigger
language plpgsql
as $$
begin
  if new.seq_id is null then
    select coalesce(max(seq_id), 0) + 1
    into new.seq_id
    from opportunities
    where tenant_id = new.tenant_id;
  end if;
  return new;
end;
$$;
```

**RLS enable + grant pattern** (de `0001_init.sql` lines 313–316, 364–370):

```sql
alter table opportunities enable row level security;
grant select, insert, update, delete on opportunities to authenticated;
```

**Function `SECURITY DEFINER` pattern** (de `0001_init.sql` lines 181–189 — `current_tenant_id`):

```sql
create or replace function current_tenant_id()
returns uuid
language sql
stable
security definer
set search_path = public
as $$
  select tenant_id from profiles where id = auth.uid()
$$;
```

**Differences to apply (per research §Pattern 2):**
- Criar tabela `tenant_sequences (tenant_id uuid pk, last_seq int)` + RLS habilitado (sem policy → só service_role acessa)
- Backfill: `insert into tenant_sequences select tenant_id, coalesce(max(seq_id),0) from opportunities group by tenant_id on conflict (tenant_id) do update set last_seq = excluded.last_seq` + linha vazia (0) para tenants sem oportunidades
- Função `next_seq_id(p_tenant_id uuid) returns int` com `INSERT ... ON CONFLICT DO UPDATE ... RETURNING last_seq` (atômico)
- **Drop e recriar** trigger `set_opportunity_seq_id` — desta vez SEMPRE sobrescreve (`new.seq_id := next_seq_id(...)` sem `if null`) para bloquear cliente que forje `seq_id`
- Naming pattern: prefixo `trg_opportunities_*` para triggers, `set_*` ou descritivo para functions (matches 0001/0004 style)

  const parsed = opportunityInputSchema.safeParse(input);
  if (!parsed.success) {
    const flat = parsed.error.flatten();
