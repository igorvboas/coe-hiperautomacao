Client Component (`'use client'`). Recebe `{ opportunities }`. Estado local com `optimisticOpps` (cópia do array) pra atualização imediata na UI.

```tsx
const [opps, setOpps] = useState(opportunities);
const [isPending, startTransition] = useTransition();
```

Renderiza:
- Wrapper horizontal scrollable
- 8 `<KanbanColumn>` lado a lado, agrupando opps por status

`DndContext` cobre tudo. `onDragEnd`:
- Lê `active.id` (opportunity id) e `over.data.current.status` (destino)
- Se status mudou, otimisticamente atualiza `setOpps` (move o card pra nova coluna)
- Chama `updateOpportunityStatus(id, newStatus)` dentro de `startTransition`
- Se a Server Action retornar `{ ok: false }`, faz rollback (restaura opps anterior) + toast erro

`PointerSensor` com `activationConstraint: { distance: 5 }` evita conflito com click.

Status order do mockup: `['novo','em_analise','planejamento','backlog','desenvolvimento','homologacao','producao','concluido']`. Cores extraídas de STATUS_MAP.
