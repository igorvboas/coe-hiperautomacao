<task type="auto">
  <name>Task 5: WizardShell (orquestrador)</name>
  <files>components/opportunities/wizard/WizardShell.tsx</files>
  <action>
Client Component principal. Props: `{ mode: 'create' | 'edit'; initialData?: Partial<OpportunityInput>; opportunityId?: string }`.

State:
- `formData: WizardFormData` (useState)
- `currentStep: number`
- `errors: Record<string, string>` (do step atual)
- `submitting: boolean`

Layout (similar ao .fov do mockup):
- Overlay full-screen
- Painel lateral à direita (~760px)
- Header gradient azul com título "Nova Oportunidade" ou "Editar Oportunidade", ícone, X close
- `<StepsNav>`
- `<div className="fpbody">` renderiza step atual via switch
- Footer: botão "Cancelar" (esquerda), "Anterior" + "Próximo" / "Finalizar" (direita)

Lógica:
- Anterior: decrementa step (sem validação)
- Próximo: roda `validateStep(currentStep, formData)`; se ok, incrementa; senão seta errors
- Finalizar (último step): chama `createOpportunity` ou `updateOpportunity`; em sucesso redireciona; em erro mostra toast/inline
- X / overlay click / ESC: `router.back()` (descarta)

Step list depende de `formData.source` (definido no step 0). Antes de escolher tipo, só TipoStep está acessível.
  </action>
  <verify>tsc passa</verify>
  <acceptance_criteria>
    - Client component
    - State completo do form
    - Navegação entre steps
    - Submit chama action certa por mode
    - Close volta sem persistir
  </acceptance_criteria>
</task>
