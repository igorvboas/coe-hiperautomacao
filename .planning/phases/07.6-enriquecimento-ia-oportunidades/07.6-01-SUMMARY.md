### Task 3 — Migration `0010_ai_enrichment.sql` (commit `e822dcb`)

Arquivo `supabase/migrations/0010_ai_enrichment.sql` (90 linhas) com 5 blocos:

1. **Enum** `ai_enrichment_status` (`pending`, `enriched`, `failed`) via `do$$ ... if not exists` (idempotente).
2. **3 colunas em `opportunities`** via `add column if not exists`:
   - `ai_enrichment_status ai_enrichment_status not null default 'pending'`
   - `ai_enrichment_error text` (nullable)
   - `ai_enriched_at timestamptz` (nullable)
3. **CHECK** `opportunities_ai_error_len_chk` (max 1000 chars) — mitiga `T-07.6-A-04` (DoS via repeated long error messages).
4. **Índice parcial** `opportunities_ai_pending_idx` em `(created_at) WHERE ai_enrichment_status = 'pending'` — suporta queries de monitoring sem custo em rows enriched/failed.
5. **DROP + CREATE** da view `opportunities_with_score` com `security_invoker = true` + `grant select to authenticated`. As 3 colunas novas entram automaticamente via `o.*`. Padrão idêntico ao das migrations 0008/0009 (mitiga `T-07.6-A-02`: ERROR 42P16 ao tentar `CREATE OR REPLACE` quando `o.*` reordena colunas).

`opportunity_score()` permanece intocado — chamada com 4-args `(esforco, complexidade, tempo, objetivo)` exatamente como em 0001/0008/0009.
