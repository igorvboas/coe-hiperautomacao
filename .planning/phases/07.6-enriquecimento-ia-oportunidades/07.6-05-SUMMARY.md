## Decisions Made

- **Default neutro literal `objetivo: 3` no submit:** `objetivo` é required no `PublicSubmitInput` type (actions.ts) e não pode ser removido sem mudar a interface do Server Action. Submit envia literal `3` como placeholder — RPC `create_public_opportunity` aceita; IA sobrescreve via UPDATE pós-INSERT. Alternativa rejeitada: tornar `objetivo` opcional no type — exigiria mudança em `lib/opportunities/actions.ts` que está fora de escopo do Plan 05 (Plan 03 já definiu o type).
- **Tipos CriterioValor / CriterioKey / CRITERIOS / BeneficioKey / BENEFICIOS removidos:** após remover os render branches, essas declarações viraram dead code dentro do arquivo. Não eram exportadas — remoção segura. Typecheck confirmaria, mas remoção proativa evita lint warnings futuros.
- **Imports DynamicList + ScorePreview removidos:** usados apenas pelos render branches removidos. Typecheck `noUnusedLocals` (não habilitado neste projeto) flagaria; remoção proativa.
- **Test via regex source-code:** PublicForm é client component com hooks + widget externo (Turnstile). Renderizar via vitest exige jsdom + @testing-library + setup de mock Turnstile — desproporcional ao risco para invariantes estruturais. Pattern já usado em `tests/wizard/state.test.ts` (Plan 04) — comportamento via funções puras / regex; UAT manual cobre o E2E.
- **Strip de comentários no test de FormState:** o type body contém um comentário multi-linha listando os campos removidos (para documentação no source). Test 3 faz strip de `// ...` antes do regex de detecção — evita falso-positivo da própria documentação anti-regressão.
- **Type body NÃO contém `?` opcionais residuais:** decisão de remover os campos *completamente* em vez de marcá-los opcionais — força typecheck a flagar qualquer referência leftover; mantém o type honesto sobre o que o form coleta.
