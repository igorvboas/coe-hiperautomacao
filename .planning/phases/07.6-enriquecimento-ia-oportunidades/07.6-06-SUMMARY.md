## Decisions Made

- **Server component puro (sem `'use client'`)** para AiEnrichmentBadge — render condicional sem hooks; safe de usar tanto em Server quanto em Client Component pai. Reduz JS bundle no client.
- **`status='enriched'` renderiza `null`** (silêncio) em vez de badge verde "Enriquecido" — sinal positivo é os campos populados nas tabs, badge extra seria ruído visual. Documentado no JSDoc do componente.
- **`title` attribute carrega o erro técnico** no badge 'failed' — tooltip nativo do browser, sem dependência de lib de tooltip (radix-popover etc.). Acessível via teclado (focus → tooltip aparece após delay padrão do OS). Trade-off aceito: estilo do tooltip varia entre OS — bom o suficiente para feedback técnico ao admin.
- **`environmentMatchGlobs` + pragma `// @vitest-environment jsdom`** como redundância — pragma sobrevive a migração futura para `test.projects` (vitest deprecou environmentMatchGlobs). Defesa em camadas.
- **Setup file `dom-matchers.ts` com import condicional** baseado em `typeof window` — evita poluir workers node-only (security/ai/opportunities/wizard/public-form tests) com matchers DOM que falhariam fora de jsdom. Single setup file, dispatch por ambiente.
- **`AiEnrichmentStatus = Opportunity['ai_enrichment_status']`** (alias, não redeclaração) — single source of truth = `lib/database.types.ts` (gen:types). Se o enum mudar no DB, basta `gen:types` + tudo propaga.
- **NÃO adicionei Realtime nem polling** para auto-atualizar o badge. RESEARCH §"Open Question 2" recommenda explicitamente "NÃO polling"; user que voltar à lista 1 min depois já vê 'enriched' (enrichment é fast ~5-10s). Aceito como T-07.6-F-03 (low risk).
- **NÃO criei rota POST `/api/opportunities/[id]/enrich`** para re-trigger manual. RESEARCH: "não há requisito disso no escopo".
- **NÃO modifiquei `AutomacaoTab.tsx`, `OpportunityDetail.tsx`, `EditButton.tsx`** — edit path já funciona via EditButton → WizardShell mode='edit' → AutomacaoStep + PriorizacaoStep (preservados em Plan 04). AI-ADMIN-01 satisfeito sem touch adicional.
